# 新手开始前先看这篇

如果你是第一次在服务器上用 Docker 部署项目，建议先看完这篇再去看每个项目的单独教程。

## 一、你至少要准备什么

部署任何一个 Docker 项目，通常都需要下面这些东西：

- 一台 Linux 服务器
- 服务器可以正常联网
- 你有服务器的 SSH 登录权限
- 服务器已经安装好 Docker
- 最好也安装好 Docker Compose
- 如果项目要通过域名访问，你还需要一个已经解析到服务器的域名

## 二、最常用的目录规划

为了后面维护方便，建议统一把项目放到 `/opt/docker/` 下面。

例如：

```bash
/opt/docker/
├── uptime-kuma/
├── nginx-proxy-manager/
├── searxng/
├── openlist/
└── ...
```

这样做的好处是：

- 所有项目都在一个地方
- 以后备份更方便
- 查找配置和数据目录更方便
- 后续迁移服务器更轻松

## 三、一个最基本的部署流程

几乎所有项目都可以按下面这个流程来：

### 第一步：创建目录

```bash
mkdir -p /opt/docker/项目名
cd /opt/docker/项目名
```

### 第二步：创建 `compose.yml`

把项目对应的 compose 配置写进去。

### 第三步：启动项目

```bash
docker compose up -d
```

### 第四步：检查是否启动成功

```bash
docker ps
```

### 第五步：看日志

```bash
docker logs -f 容器名
```

### 第六步：浏览器访问

如果项目开放端口，就访问：

```text
http://服务器IP:端口
```

如果你已经配置了反向代理和 HTTPS，也可以直接访问域名。

## 四、什么是端口映射

比如下面这行：

```yaml
ports:
  - "3001:3001"
```

前面的 `3001` 是服务器对外开放的端口。
后面的 `3001` 是容器内部程序真正监听的端口。

如果你改成这样：

```yaml
ports:
  - "8888:3001"
```

那就表示：

- 容器内部还是监听 `3001`
- 但你访问服务器时要用 `8888`

也就是说，浏览器里要打开：

```text
http://服务器IP:8888
```

## 五、什么是数据持久化

很多项目都需要把数据保存到宿主机，不然删容器后数据就可能没了。

例如：

```yaml
volumes:
  - ./data:/app/data
```

这表示：

- 宿主机当前目录下的 `data` 文件夹
- 挂载到容器里的 `/app/data`

这样容器里的数据就会写到宿主机上，重建容器也不容易丢失。

## 六、什么是反向代理

反向代理的意思是：

- 你不直接访问 `服务器IP:端口`
- 而是访问域名，例如 `https://status.xxx.com`
- 再由 Nginx Proxy Manager 这类工具转发到实际容器端口

这样做的好处：

- 地址更好记
- 可以统一走 HTTPS
- 可以隐藏实际端口
- 多个项目都能共用 80 和 443 端口对外访问

## 七、推荐你先部署哪个项目

如果你是新手，建议按这个顺序练手：

1. `Uptime Kuma`：最容易上手
2. `Nginx Proxy Manager`：学会域名和 HTTPS
3. `OpenList`：学会数据目录挂载
4. `SearXNG`：学会多服务 compose

## 八、更新一个项目的标准步骤

进入项目目录后执行：

```bash
docker compose pull
docker compose up -d
```

如果你担心出问题，更新前先备份数据目录。

## 九、更新前为什么建议备份

因为有些项目升级后会：

- 改数据库结构
- 改配置格式
- 需要重新初始化某些内容

最稳妥的办法就是先备份挂载目录。

例如：

```bash
tar -czvf backup.tar.gz /opt/docker/项目名/data
```

## 十、遇到问题时先看哪里

建议按顺序排查：

1. `docker ps` 看容器是不是正在运行
2. `docker logs -f 容器名` 看报错
3. 检查端口是否映射正确
4. 检查防火墙和安全组
5. 检查域名是否解析到服务器
6. 检查反向代理配置是否正确

## 十一、建议你养成的好习惯

- 每个项目单独一个目录
- 每个项目都保留 `compose.yml`
- 每个项目都明确数据目录
- 更新前先备份
- 不要随便删除数据目录
- 重要项目使用域名和 HTTPS

## 十二、下一步怎么看文档

你可以按这个顺序继续看：

1. 先看 [Docker 常用命令手册](./docker-commands.md)
2. 再看你要部署的项目教程
3. 如果要统一管理域名，优先看 `Nginx Proxy Manager` 教程
