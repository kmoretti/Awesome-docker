# 备份与迁移教程

这篇文档讲的是：

- 怎么备份 Docker 项目
- 怎么迁移到另一台服务器
- 哪些东西最重要不能丢

## 一、最重要的原则

真正重要的通常不是容器本身，而是：

- `compose.yml`
- 环境变量文件，例如 `.env`
- 挂载到宿主机的数据目录
- 配置文件

## 二、为什么不需要备份容器本身

因为容器一般可以重新创建。

真正不能丢的是：

- 项目数据
- 登录信息
- 数据库
- 配置文件
- 上传的文件

## 三、通用备份思路

例如你的项目目录是：

```bash
/opt/docker/uptime-kuma/
```

你通常要备份：

- `compose.yml`
- `data/`

例如：

```bash
tar -czvf uptime-kuma-backup.tar.gz /opt/docker/uptime-kuma
```

## 四、迁移到新服务器的标准流程

### 1. 新服务器安装 Docker

### 2. 把旧服务器项目目录打包

### 3. 传到新服务器

### 4. 解压到相同目录

### 5. 重新执行：

```bash
docker compose up -d
```

## 五、哪些项目最需要重视数据目录

例如：

- `Uptime Kuma`
- `OpenList`
- `Nginx Proxy Manager`
- `Ech0`
- `ChronoFrame`

因为这些项目通常会保存：

- 用户账号
- 配置
- 数据库
- 文件内容

## 六、迁移后第一件事做什么

建议先检查：

```bash
docker ps
docker logs -f 容器名
```

然后再用浏览器测试是否正常。

## 七、常见问题

### 1. 新服务器启动后页面打不开

先检查：

- 端口有没有开放
- 反向代理是否重新配置
- 数据目录权限是否正常

### 2. 登录信息没了

大概率是数据目录没带过去，或者挂载路径变了。
