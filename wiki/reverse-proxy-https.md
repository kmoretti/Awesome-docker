# 反向代理与 HTTPS 教程

这篇文档讲的是：

- 为什么要用域名访问 Docker 项目
- 为什么要做 HTTPS
- 怎么用 `Nginx Proxy Manager` 管理这些项目

## 一、为什么不要总是直接访问 IP:端口

直接访问：

```text
http://服务器IP:3001
```

虽然能用，但有几个问题：

- 不好记
- 不美观
- 不方便统一管理
- 没有 HTTPS 时不安全

## 二、什么是反向代理

反向代理可以理解成：

- 用户访问域名
- 反向代理接收请求
- 再把请求转发给真正运行项目的端口

例如：

- 访问 `https://status.your-domain.com`
- 实际转发到 `http://127.0.0.1:3001`

## 三、为什么推荐 Nginx Proxy Manager

因为它：

- 有网页后台
- 对新手友好
- 可以申请 HTTPS 证书
- 可以同时管理多个项目域名

## 四、最常见的使用方式

例如你已经部署了：

- `Uptime Kuma`，端口 `3001`
- `OpenList`，端口 `5244`
- `SearXNG`，端口 `8080`

你可以在 Nginx Proxy Manager 中分别创建：

- `status.your-domain.com -> 3001`
- `list.your-domain.com -> 5244`
- `search.your-domain.com -> 8080`

## 五、HTTPS 是怎么来的

最常见做法是通过 Let's Encrypt 自动申请免费证书。

前提通常是：

- 域名已经解析到你的服务器
- 80 和 443 端口可被公网访问

## 六、典型流程

### 1. 域名解析到服务器 IP

### 2. 启动 Nginx Proxy Manager

### 3. 新建 Proxy Host

### 4. 填写域名和目标端口

### 5. 申请 SSL 证书

### 6. 开启强制 HTTPS

## 七、常见问题

### 1. 域名访问不了

请检查：

- DNS 是否解析正确
- 域名是否生效
- 反向代理是否配置正确

### 2. 证书申请失败

请检查：

- 80 和 443 是否开放
- 域名是否真的指向当前服务器
- 没有其他服务占用 80 / 443

### 3. 页面一直 502

通常说明反向代理转发的目标地址或端口写错了。
