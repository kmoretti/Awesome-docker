# 常见故障排查教程

这篇文档适合在“项目起不来、网页打不开、域名不通、数据异常”时使用。

## 一、先记住排查顺序

不要一上来就删容器、删镜像、删数据。

建议按下面顺序排查：

1. 看容器是否启动
2. 看日志是否报错
3. 看端口是否监听
4. 看防火墙和安全组
5. 看反向代理配置
6. 看域名解析
7. 看数据目录权限

## 二、第一步：看容器在不在运行

```bash
docker ps
```

如果没有运行，再看所有容器：

```bash
docker ps -a
```

## 三、第二步：看日志

```bash
docker logs -f 容器名
```

这一步通常最关键。

## 四、第三步：检查 compose 文件有没有写错

```bash
docker compose config
```

## 五、第四步：检查端口是否监听

```bash
ss -tlnp
```

## 六、第五步：检查反向代理

如果你是通过域名访问的，请检查：

- 域名是否解析对了
- Nginx Proxy Manager 是否转发到正确端口
- 证书是否正常

## 七、第六步：检查数据目录

很多项目的问题都和挂载目录有关。

请确认：

- 路径有没有写错
- 目录是否存在
- 权限是否正常

## 八、常见错误场景

### 1. 容器不断重启

通常是：

- 环境变量没填对
- 配置文件错误
- 程序启动即报错

### 2. 页面 502

通常是反向代理目标端口写错了。

### 3. 更新后异常

先不要删除数据。
先看日志，再考虑回滚或恢复备份。

### 4. 数据丢失

先检查是不是没有做数据挂载。
如果没挂载，重建容器后很容易丢数据。
